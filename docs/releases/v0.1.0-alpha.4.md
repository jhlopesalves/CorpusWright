# CorpusWright v0.1.0-alpha.4

Theme: smarter artefact review and safer extraction-noise cleaning.

## Highlights

- Repeated artefact review is more useful without becoming automatic deletion.
- Exact and normalised whole-line candidate families can become structured removal rules.
- Page-aware repeated artefact evidence can produce page-zone scoped removal rules when reliable PDF/OCR page metadata is available.
- Deterministic text/noise profiling helps review candidates and powers an opt-in obvious extraction/OCR noise cleaner.
- Windows and application icon packaging now uses the transparent icon source, with no white matte outside the blue rounded square.

## Added

- Structured custom removal rules for whole-line literal matches and normalised whole-line family matches.
- Page-aware document metadata for page-local line positions.
- Cached PDF and OCR page text preservation for page-aware downstream workflows.
- Page-zone scoped removal rules for page top, page bottom, and page top-or-bottom matches.
- Candidate text/noise profiles with advisory labels for natural text, headings, page labels, tables, formulae, code-like lines, extraction noise, and ambiguous text.
- Opt-in obvious extraction/OCR noise cleaning for overwhelmingly likely full-line extraction, OCR, and markup junk.
- Multilingual heading and page-label heuristics for candidate review.

## Changed

- Conservative page-edge candidate promotion keeps candidates as whole-line rules unless reliable page metadata shows only top and/or bottom occurrences.
- Exact and normalised repeated artefact promotion now records structured rule intent instead of relying only on legacy substring removals.
- Generated TypeScript bindings include the current cleaning, repeated artefact, and removal-rule types.
- Saved configurations without newer cleaning fields continue to load through defaults.
- CI/build validation covers Rust checks, generated TypeScript bindings, release version consistency, and the frontend build.

## Fixed

- The Windows app, shortcut, taskbar, and package icon assets are regenerated from the transparent icon source.
- The icon no longer carries an opaque white square or matte around the rounded blue app mark.

## Known Limitations

- The classifier is deterministic and heuristic; it does not use a language model or model-backed classifier.
- Short or ambiguous text can remain ambiguous.
- Removed obvious-noise line counts are not yet reported.
- Page-zone rules require reliable page metadata.
- Flat text paths do not infer page zones from blank lines or file position.
- Windows may cache old desktop, Start menu, or taskbar icons after reinstall. Users who still see an old icon may need to recreate shortcuts, reinstall the app, restart Explorer, or refresh the icon cache.

## Validation

Run the release validation set before tagging:

```bash
cargo fmt
cargo clippy -p corpuswright-core --all-targets -- -D warnings
cargo clippy -p corpuswright-desktop --all-targets -- -D warnings
cargo test -p corpuswright-core -- --test-threads=1
cargo check -p corpuswright-core
cargo check -p corpuswright-desktop --all-targets
cargo run -p corpuswright-core --example export_ts
npm --prefix apps/desktop run build
node scripts/check-release-consistency.cjs
git diff --check
```

Release packaging should also run the Tauri release build used by `.github/workflows/release.yml`.
